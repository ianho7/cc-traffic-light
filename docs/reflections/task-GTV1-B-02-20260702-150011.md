- Task: GTV1-B-02 Implement settings window shell and navigation scaffold
- Encountered Problem: The V1 settings UI needed to stay pure Win32, but there was no existing window shell for a non-taskbar configuration surface.
- Thought Process: I avoided child-control complexity and started with a dedicated top-level Win32 window that paints a static control-panel layout directly.
- Options Considered: Delay settings until tray exists; build child controls first; build a custom-painted shell first and add interactive controls later.
- Chosen Solution: Build a custom-painted settings shell first.
- Rationale: It establishes window lifecycle, layout direction, and overview content immediately while keeping the first implementation slice small.
