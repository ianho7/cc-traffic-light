- Task: TSM-E-02 建立 Tauri 前端 design tokens
- Encountered Problem: 原有样式更像临时 scaffold，颜色、边框、圆角、间距和字体角色没有统一 token，后续页面扩展会继续散。
- Thought Process: 我直接把 demo/spec 里稳定的视觉语义抽成 CSS variables，覆盖 app background、panel、text、border、status color、spacing、radius 和 font 角色，但不引入复杂主题系统。
- Options Considered: 继续把样式写死在组件类名里；提前做完整主题切换系统；用一套轻量 CSS variables 先收口视觉基线。
- Chosen Solution: 在 `taskbar-settings-tauri/src/styles.css` 顶部建立全局 tokens，并让页面壳、section、badge、row、toggle 等公共样式全部消费这些 token。
- Rationale: 这满足了 E-02 对 design tokens 的要求，也避免在还没完成 settings 迁移时过早引入额外主题架构。
