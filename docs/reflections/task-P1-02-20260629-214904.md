- Task: P1-02 在 `main.rs` 中完成窗口类注册、窗口创建、显示与消息循环
- Encountered Problem: `windows 0.58` 的 API 细节和常见示例略有差异，第一次实现时在 `CreateWindowExW` 和消息链路上出现签名偏差。
- Thought Process: 先把最小流程写出来，再用 `cargo check` 反推真实签名，而不是凭记忆硬写更多封装层。
- Options Considered: 引入更高层封装；继续猜测 API；保留最小实现并根据编译器错误修正。
- Chosen Solution: 在 `main.rs` 中保留直接的 Win32 调用路径，并根据编译反馈修正 `CreateWindowExW` 和消息循环细节。
- Rationale: 现阶段最有价值的是一条真实可编译的 Win32 链路，不是抽象层。
