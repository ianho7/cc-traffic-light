- Task: SSM-E-04 把 tray 菜单文案改为通过 i18n 层输出
- Encountered Problem: 旧 tray 菜单是 Win32 `AppendMenuW` 的英文硬编码，和新的国际化路径不一致。
- Thought Process: 在保持 Win32 tray 宿主不变的前提下，只替换文案来源，让菜单文本由 `Localizer` 生成。
- Options Considered: 保持硬编码直到语言切换 UI 完成；先把菜单字符串切到共享 i18n 层。
- Chosen Solution: `tray_icon.rs` 改为根据当前 config 的 locale 输出菜单文案。
- Rationale: 这样即便语言切换 UI 还没接完，tray 菜单的文本来源已经统一，不再需要二次重构。
