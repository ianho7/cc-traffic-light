- Task: GHI-B-04 implement in-memory merge planning
- Encountered Problem: dry-run 不能写盘，但必须提前暴露“哪些事件会新增、哪些会更新”的计划摘要。
- Thought Process: 先在内存里构造受管 hooks，再把现有条目拆成 managed 和 other 两类。
- Options Considered: dry-run 只输出成功/失败；直接生成完整 JSON diff；输出事件级 add/update/unchanged 摘要。
- Chosen Solution: 为每个 lifecycle event 生成事件级摘要，并在内存中完成 merge 规划。
- Rationale: 这种摘要既足够 review，也不会泄露完整用户配置内容。
