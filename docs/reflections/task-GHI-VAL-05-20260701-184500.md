- Task: GHI-VAL-05 validate fixture restore returns to pre-apply state
- Encountered Problem: restore 如果只“看起来成功”但内容没回到 apply 前，就不算真正可回滚。
- Thought Process: 必须在 apply 之后立即做 restore，形成成对验证。
- Options Considered: 只看 restore 命令返回；只看 backup；直接读取恢复后的 fixture 内容。
- Chosen Solution: 对 fixture 执行 apply 后再 restore，并检查恢复结果。
- Rationale: 这样 restore 的证据来自文件状态，而不是脚本自述。
