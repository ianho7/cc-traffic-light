- Task: GHI-C-01 implement predictable backup before write
- Encountered Problem: `apply` 一旦写坏真实用户配置，没有可预测备份就很难回滚。
- Thought Process: backup 路径必须稳定，且不能因为多次运行轻易覆盖第一次的原始状态。
- Options Considered: 每次覆盖同一个备份；时间戳备份一堆文件；固定 backup 路径并单独记录元数据。
- Chosen Solution: 使用固定 backup 文件加 `meta` 文件记录 `originalExisted`。
- Rationale: 这同时覆盖“原文件存在”和“原文件不存在”两条 restore 语义。
