- Task: TSM-D-02 选择 React 并建立最小页面壳子
- Encountered Problem: Tauri settings 需要尽快呈现 6 页结构，但这一步还不应该混入真实宿主数据。
- Thought Process: 我优先搭出一个最小 React shell，用统一侧边导航和页面切换把 Overview/General/Monitoring/Appearance/Diagnostics/About 六页立住，再把 fake backend 接进去。
- Options Considered: 只做一个空白页；先上 React Router；先用无依赖的最小导航状态管理把 6 页结构立起来。
- Chosen Solution: 使用 React 19 + Vite，前端通过状态驱动的 6 页 shell 对齐现有 settings 页面结构。
- Rationale: 这足以验证 Tauri 前端承载页面结构的能力，同时避免在脚手架阶段增加路由库和额外噪音。
