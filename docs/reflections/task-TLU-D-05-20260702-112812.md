- Task: TLU-D-05 当真实 hook 与人工 set 不一致时定位并修正状态映射
- Encountered Problem: 初次真实 hook 验证里，失败落到 `done`，批准路径停在 `waiting`，与人工矩阵不一致。
- Thought Process: 直接检查 `hook_rules.rs`，发现 `Stop` 被过宽 waiting heuristic 劫持，失败路径又缺少 `Stop` 层面的 error 兜底。
- Options Considered: 继续做更多 UI 观察；直接修正 hook 规则。
- Chosen Solution: 收紧 `Stop` waiting 逻辑，并给 `Stop` 增加失败文本识别，同时补 `PostToolUseFailure` / `ToolUseFailure` 到 `error`。
- Rationale: 复测结果已证明问题在映射层而不是绘制层，修规则是最短路径。
