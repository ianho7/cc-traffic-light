- Task: P4-03 使用固定宽高与固定偏移调用 `MoveWindow`，先追求“稳定可见”，不追求优雅布局
- Encountered Problem: 一旦开始追求“完美尺寸”，很容易重新掉进布局优化黑洞。
- Thought Process: 用固定模块宽度、父窗口高度和固定 margin 先得到一个清晰可见的矩形。
- Options Considered: 动态计算复杂尺寸；保留原顶层窗口尺寸；使用固定宽高和保守偏移。
- Chosen Solution: 通过 `MoveWindow` 把模块移动到 `TrayNotifyWnd` 左侧的固定区域。
- Rationale: 这正符合 MVP 的“稳定可见优先”原则。
