- Task: GHI-C-03 implement minimal-risk write flow
- Encountered Problem: `.NET File.Replace()` 在当前环境下因为空备份参数不可用，第一版写入路径失败了。
- Thought Process: 既然已经有独立 backup，就不必再强行依赖 `File.Replace()` 这一层。
- Options Considered: 继续调 `File.Replace()`；直接原地 `Set-Content`；先写临时文件再替换目标。
- Chosen Solution: 采用“临时文件 + 删除旧文件 + 移动新文件”的最小风险写入路径。
- Rationale: 虽然不算强原子，但在已有独立 backup 前提下足够稳妥，也更兼容当前 PowerShell/.NET 运行时。
