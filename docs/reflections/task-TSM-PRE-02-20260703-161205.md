- Task: TSM-PRE-02 审计共享 core 的候选边界
- Encountered Problem: 现有配置、状态和运行时契约分散在多个 Rust 模块里，如果不先识别纯业务边界，后续很容易把 Win32 或 UI 细节带进 shared-core。
- Thought Process: 我对照 `app_config.rs`、`ui_state.rs`、`runtime_contract.rs`，区分出纯数据结构与平台耦合逻辑。
- Options Considered: 直接把整个 settings backend 迁出去；只迁配置结构；先标出纯 DTO 和 service 候选边界。
- Chosen Solution: 先把 `AppConfig`、`AppStatusSnapshot` 投影、settings service 和错误类型作为第一批 shared-core 候选。
- Rationale: 这样边界保守，能先服务 Tauri，而不会把 Win32 消息循环一起拖进共享层。
