- Task: TSM-E-04 明确静态 fidelity 的近似还原边界
- Encountered Problem: 如果不先写清“必须近似还原”和“允许偏离”的分界线，后续很容易把 demo 的窗口 chrome、演示脚本或像素级细节误当成必须完成项。
- Thought Process: 我把这一步限定为视觉边界定义，而不是继续扩 UI 代码；先对照 spec 和当前实现，把结构、导航、row 节奏、状态表达列为必须近似还原项，再把 titlebar、像素级对齐、演示脚本和完整截图归档列为当前允许保留差异。
- Options Considered: 只在 checklist 上简写几句；默认追求全面像素复刻；单独产出一份 fidelity 边界文档并把允许偏差写清楚。
- Chosen Solution: 新增 `docs/plan/tauri-settings-visual-fidelity-pass-1.md`，明确哪些视觉与信息层级必须贴近 demo，哪些地方为了工程稳定和当前阶段边界允许做实现调整。
- Rationale: 这样后续做 read-only integration、写路径和默认入口切换时，就不会把静态视觉任务与窗口系统、业务边界混在一起。
