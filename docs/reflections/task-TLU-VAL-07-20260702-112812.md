- Task: TLU-VAL-07 验证真实 hook smoke test 能否用开始态和结束态自动驱动 UI 更新
- Encountered Problem: 初次 smoke test 中，真实失败和 waiting 终态都不可靠，无法把该验证项判为通过。
- Thought Process: 修正 hook 规则后，重新用成功、失败、waiting 三类 prompt 复测主链路。
- Options Considered: 保持未通过等待更多人工证据；基于修正规则后的复测结果重新判定。
- Chosen Solution: 依据 `run -> done`、`run -> error`、`run -> wait -> error` 三条结果，把该验证项标记为通过。
- Rationale: 这已经证明真实 hook 可以自动把 widget 带入明确开始态和明确终态。
