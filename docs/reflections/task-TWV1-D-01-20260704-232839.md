- Task: TWV1-D-01 Design a minimal phase-1 visual config model
- Encountered Problem: The repository already had legacy appearance fields whose semantics were explicitly out of scope for this phase.
- Thought Process: I introduced a new config block that maps directly to the working phase-1 renderer instead of trying to retrofit old flags.
- Options Considered: Reuse `indicator_style`; overload `widget_size`; add a new `widget_visual.palette` block.
- Chosen Solution: Add `widget_visual.palette.{green,yellow,red,off}`.
- Rationale: Those fields are narrow, effective, serializable, and do not pretend that old no-op appearance settings now have meaning.
