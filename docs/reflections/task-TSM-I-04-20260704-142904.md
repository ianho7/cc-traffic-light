- Task: TSM-I-04 清理 taskbar-widget/Cargo.toml 中仅用于 Slint settings 的依赖与 build 脚本
- Encountered Problem: 即使运行时不再使用 Slint，`Cargo.toml` 和 `build.rs` 里的依赖与编译步骤仍会继续把它当活跃技术栈保留下来。
- Thought Process: 我先确认宿主不再引用 Slint 类型或模块，再清理依赖，最后保留 manifest 嵌入逻辑避免误伤之前修复过的 Windows loader 约束。
- Options Considered: 1. 暂时保留依赖。2. 只删运行时引用，不删 build 依赖。3. 同时删 `slint`、`slint-build`、`raw-window-handle` 和对应编译步骤。
- Chosen Solution: 移除 Slint 相关依赖，并把 `build.rs` 精简为只负责 Windows manifest。
- Rationale: 这样可以把 Slint 从宿主构建图里真正摘掉，同时保留已经验证有效的 manifest 构建约束。
