- Task: TSM-PRE-04 确认最小验证命令集合
- Encountered Problem: 清单里同时包含 Rust 和未来前端/Tauri 验证，如果不先冻结最小验证命令，阶段推进会失焦。
- Thought Process: 我按当前仓库已有能力和未来目标拆分验证：现阶段先保 Rust 编译与测试，前端命令等 Tauri 工程创建后再纳入。
- Options Considered: 立刻要求全套 `pnpm`/Tauri 验证；只保 `cargo check`；按阶段保留 `cargo check`、Rust tests、未来 `pnpm build` 和必要时 `cargo build`。
- Chosen Solution: 冻结分阶段最小验证命令集合。
- Rationale: 这保证当前每轮都能做真实验证，同时不给还不存在的前端工程强行背命令。
