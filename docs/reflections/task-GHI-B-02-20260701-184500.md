- Task: GHI-B-02 implement hooks.json read and missing-file fallback
- Encountered Problem: 用户级 `hooks.json` 可能不存在，脚本不能把“缺失”直接当成错误。
- Thought Process: 读取路径时要区分“文件不存在”和“文件存在但内容无效”两类情况。
- Options Considered: 缺失时报错；只支持已有文件；缺失时合成空 hooks 配置继续规划。
- Chosen Solution: 缺失文件时返回 `Exists = false` 和空的 `{ hooks = {} }` 结构。
- Rationale: 这既满足 dry-run，也让后续 apply/restore 能覆盖首次安装路径。
