- Task: P3-03 记录 `SetParent` 返回值、`GetLastError`、调用前后父子关系
- Encountered Problem: 若只看“有没有崩”，根本无法证明父子关系真的发生了变化。
- Thought Process: 把 `previous_parent`、`current_parent` 和 style 前后值一起记下来，形成最小审计线索。
- Options Considered: 只记录成功/失败布尔值；只记录新父句柄；记录父句柄和样式前后。
- Chosen Solution: 输出了 `success=true`、`previous_parent=0x0`、`current_parent=0x10102`、`style_before`、`style_after`。
- Rationale: 这让“已挂到任务栏层级”成为可审计事实，而不是模糊结论。
