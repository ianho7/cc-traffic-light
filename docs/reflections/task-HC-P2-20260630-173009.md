- Task: HC-P2-GATE, HC-P2-01 through HC-P2-11
- Encountered Problem: Windows PowerShell 管道向 native exe 传 stdin 时可能带 UTF-16LE 表现，最初 UTF-8 字符串读取导致人工验证 JSON 解析失败。
- Thought Process: 真实 hook 多半会传 UTF-8 JSON，但本项目的主要人工验证路径是在 Windows PowerShell 中执行，CLI 必须兼容该输入形态。
- Options Considered: 要求验证时用文件重定向；只支持 UTF-8；按字节读取并兼容 UTF-8 BOM/UTF-16LE。
- Chosen Solution: hook CLI 改为按字节读取 stdin，兼容 UTF-8、UTF-8 BOM 和 UTF-16LE；非法 JSON、非法 agent、写入失败继续返回非 0。
- Rationale: 兼容 Windows 手工验证不会扩大运行时依赖，也能让 hook CLI 在真实接入前更容易被诊断。
