- Task: TSM-F-03 确认关键页面都显示真实数据
- Encountered Problem: 虽然 Tauri UI 已经有 named pipe live path，但 backend 仍默认允许 fake fallback，这会削弱“Overview / Diagnostics / About / source cards 已经不依赖假数据”的结论。
- Thought Process: 我把这一步集中在“默认数据来源”而不是继续扩字段：Overview 和 Diagnostics 已经通过 `get_snapshot` / `get_settings` 轮询真实 read model，About 已补齐真实 version/config path/runtime metadata，剩下要做的是把 fake backend 降级成显式开发开关，而不是默认 fallback。
- Options Considered: 保留当前优先 live、失败即 fake；直接彻底删除 fake backend；把 fake backend 收成 `CC_TRAFFIC_LIGHT_TAURI_FAKE_BACKEND=1` 才能启用的显式路径。
- Chosen Solution: 更新 `taskbar-settings-tauri/src-tauri/src/lib.rs`，默认要求宿主 named pipe 可用，仅在设置 `CC_TRAFFIC_LIGHT_TAURI_FAKE_BACKEND=1` 时才允许 fake mode；同时给前端补 bootstrap 错误态，并保留 About 的真实只读元数据展示。
- Rationale: 这样正常运行路径下 Tauri settings 已经不再依赖假数据，但开发期仍保留一个显式可控的孤立 UI fallback，既满足 F-03，又不破坏前期 scaffold 的可用性。
