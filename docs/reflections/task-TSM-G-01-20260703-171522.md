- Task: TSM-G-01 接通当前真实可写项
- Encountered Problem: Tauri UI 的写路径此前只覆盖了 General、Monitoring 和 Diagnostics 的一部分动作，Appearance 页虽然展示了真实字段，但仍停留在只读状态，导致正式可写项没有闭环。
- Thought Process: 我先回查旧 Slint settings 的真实交互语义，确认 `ui_theme`、`indicator_style`、`widget_size` 是 cycle，`show_labels` 和 `reduced_motion` 是 toggle，然后在 Tauri 侧按同样语义接回现有 `saveSettings` / `notifySettingsApplied` 路径，而不是自造新的表单模型。
- Options Considered: 继续把 Appearance 保持只读；一次性重构成完整表单提交；按旧 UI 语义逐项接通现有写路径。
- Chosen Solution: 更新 `taskbar-settings-tauri/src/App.tsx`，让 General、Monitoring、Appearance、Diagnostics 四页的当前真实可写项都通过 `updateConfig` 或 `requestRefresh` 进入现有宿主保存/刷新链路。
- Rationale: 这满足了 G-01 对“当前真实可写项”覆盖范围的要求，并且保持了与旧 Slint UI 一致的交互语义，后续再继续处理 shared-core 写边界和即时应用细化即可。
