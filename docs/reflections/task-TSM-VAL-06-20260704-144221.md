- Task: TSM-VAL-06 Overview 和 Diagnostics 显示真实运行状态，而不是静态演示数据
- Encountered Problem: 当前 Tauri 后端保留 fake backend 作为开发兜底，因此需要明确证明 Overview 和 Diagnostics 在正常路径下消费的是 live host 状态，而不是静态演示对象。
- Thought Process: 我把“真实状态”拆成三段证据链：前端页面字段绑定、Tauri invoke 命令、宿主 named-pipe 响应。只要三段都对上，Overview 和 Diagnostics 就不可能停留在 demo 数据。
- Options Considered: 1. 依赖现有代码阅读。2. 只做 GUI 手工查看。3. 结合字段覆盖审计与 live named-pipe 请求。
- Chosen Solution: 用 `verify-settings-coverage.ps1` 校验 `snapshot.*` / `source.*` / `settings.diagnostics.last_manual_refresh_at` 等字段确实被 Overview 和 Diagnostics 渲染，再用 live read-model 脚本确认 `get_snapshot` / `get_settings` / `request_refresh` 的真实宿主回路。
- Rationale: 这样能直接回答“页面是不是在显示真实运行状态”这个问题，而不是只证明后端或只证明静态页面。
