- Task: TSM-D-04 落地根级脚本与工作区文件
- Encountered Problem: Tauri 工程不是独立仓库，若没有根级 PNPM 工作区和脚本，后续构建命令会散落且难以交接。
- Thought Process: 我把 Node workspace 的入口放在仓库根目录，同时补齐 ignore 规则，保证 Rust 与前端两套构建产物不会互相污染。
- Options Considered: 让前端工程自成孤岛；只写局部 package.json；同时补 `pnpm-workspace.yaml`、根级 `package.json` 和 ignore 规则。
- Chosen Solution: 新增根级 `pnpm-workspace.yaml`、根级 `package.json` 脚本，并补充 `.gitignore` 以覆盖 Node、dist、Tauri gen 和 tsbuildinfo 产物。
- Rationale: 这样后续构建入口统一，也减少把生成物误提交到仓库的风险。
