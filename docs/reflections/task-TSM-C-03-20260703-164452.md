- Task: TSM-C-03 设计宿主与 Tauri 的通信协议
- Encountered Problem: 如果先起 Tauri 工程再反推协议，前后端和宿主很容易各自长出一套命令名与 DTO，后面整合成本会更高。
- Thought Process: 我先从 checklist 明确要求的五个命令反推最小协议面，再把协议常量、DTO 和 envelope 放进 shared-core，确保 fake backend 和未来真实宿主 adapter 用同一套定义。
- Options Considered: 先只写文档；直接在 Tauri backend 里写裸 command；把命令文档和 Rust 协议类型同时落到 shared-core。
- Chosen Solution: 新增 `docs/plan/tauri-settings-ipc-contract.md` 与 `shared-core::tauri_ipc`，固定 `get_snapshot`、`get_settings`、`save_settings`、`request_refresh`、`notify_settings_applied` 五个命令。
- Rationale: 这样协议先于实现收敛，后续接入真实宿主 pipe 时不需要再重命名命令或重做 DTO。
