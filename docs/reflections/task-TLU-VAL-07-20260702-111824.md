- Task: TLU-VAL-07 验证真实 hook smoke test 是否能用开始态和结束态自动驱动 UI 更新
- Encountered Problem: 成功路径可以自动结束，但失败路径和批准路径没有稳定落到正确终态。
- Thought Process: 把 smoke test 的通过标准从“看见变化就算过”收紧到“必须进入正确终态”。
- Options Considered: 因为有 `run -> done` 就勾通过；按更严格标准保持未通过。
- Chosen Solution: 保持该验证项未通过。
- Rationale: 如果失败请求最终仍显示 `done`，那么真实状态语义仍然不可信，不能把 smoke test 判为通过。
