- Task: AUD-2-07 Codex hooks 异常输入与只读文件
- Encountered Problem: 初次只读 fixture 发现脚本通过 `Remove-Item -Force` 替换只读文件，清除了只读属性并继续写入，存在静默修改用户配置的风险。
- Thought Process: 对只读用户配置应 fail closed；错误必须包含路径，且 apply 前不能创建 backup 或替换原文件。
- Options Considered: 接受 Windows `-Force` 行为；尝试恢复只读属性；apply 前明确拒绝只读文件。
- Chosen Solution: 在 `install-codex-hooks.ps1` apply 路径检测 `FileAttributes.ReadOnly`，抛出明确错误并保持 hooks.json 不变。
- Rationale: 保护用户配置，避免安装动作绕过用户设置的只读保护。
- Evidence: 隔离 `target/audit-readonly-hooks` fixture；脚本退出码 1，输出 `hooks.json is read-only and was not modified`，文件仍存在且属性仍为 ReadOnly；非法 JSON、空文件和缺失目录结果已有此前 fixture evidence。
- Status: completed
