- Task: TLU-C-05 用 debug set 验证 `error` 的颜色、形态和可读性
- Encountered Problem: `error` 是最高优先级状态，需要确认它在最终 UI 中足够醒目但不失真。
- Thought Process: 直接用单任务 `error` 检查红色状态灯、紧凑标签和计数是否都成立。
- Options Considered: 只看初始污染状态中的 `err 5`；在隔离状态下验证 `err 1`。
- Chosen Solution: 使用隔离状态验证 `err 1`。
- Rationale: 隔离后的结果更能证明当前红色错误态就是 UI 设计本身，而不是历史脏数据产物。
