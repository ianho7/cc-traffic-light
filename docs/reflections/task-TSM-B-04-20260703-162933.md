- Task: TSM-B-04 抽出 settings DTO / view model 投影
- Encountered Problem: `AppStatusSnapshot` 直接暴露给新 UI 会把内部 Rust 枚举和 runtime 结构耦合到外层表示层。
- Thought Process: 我先在 shared-core 中补一层只读投影，把状态枚举和值转成更适合跨边界传输的字符串化视图。
- Options Considered: 让 Tauri 直接消费内部结构；把 DTO 定义留到 IPC 阶段；先补最小 `SettingsReadModel` 和 `StatusSnapshotView`。
- Chosen Solution: 新增 `SettingsReadModel`、`StatusSnapshotView` 和 `SourceStatusView`，并实现从 `AppStatusSnapshot` 到视图 DTO 的投影。
- Rationale: 这样后续不管是 Tauri command 还是本地 IPC，都不需要直接绑定宿主内部运行时结构。
