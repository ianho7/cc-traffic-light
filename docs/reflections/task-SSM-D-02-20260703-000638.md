- Task: SSM-D-02 实现 AppStatusSnapshot 到 Slint view model 的状态推送
- Encountered Problem: 没有统一的 snapshot 推送逻辑，Slint 窗口即使存在也无法反映主运行时状态。
- Thought Process: 复用当前统一的 `AppStatusSnapshot`，把状态投影逻辑集中到 `settings_slint.rs`，并通过 event-loop safe 方式跨线程更新。
- Options Considered: 在 main.rs 内零散设置 Slint 属性；在 Slint runtime bridge 中集中做状态映射。
- Chosen Solution: 在 `settings_slint.rs` 中实现 `apply_state()`，并通过 `upgrade_in_event_loop()` 推送 snapshot。
- Rationale: 这样状态桥接层保持单点收口，也减少了主消息循环里的 UI 细节。
