- Task: TSM-E-01 建立 Tauri 前端静态布局
- Encountered Problem: 现有 Tauri scaffold 只有占位式卡片和列表，虽然能跑通命令回路，但结构、节奏和信息层级都还离 Nothing Signal Console 基线很远。
- Thought Process: 我先不扩展新的数据字段，也不引入新的交互，而是把页面壳、主布局、导航、分区和 6 页内容框架先按 demo 的层级重新排布，确保后续只需要把真实字段继续填入。
- Options Considered: 继续沿用当前通用管理台式布局；直接追求像素级复刻 HTML demo；先做贴近 demo 的静态骨架并保留现有数据绑定与保存逻辑。

- Rationale: 这样能先把 UI 迁移的视觉和信息架构立住，同时不混入新的业务边界，符合“先做结构，再接业务”的 checklist 顺序。
