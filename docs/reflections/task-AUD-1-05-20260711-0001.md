- Task: AUD-1-05 安装路径与 hook command 契约
- Encountered Problem: 已安装程序位于 `C:\Users\admin\AppData\Local\Programs\CC Traffic Light`，但用户 hooks.json 仍引用旧的 `C:\Users\admin\AppData\Local\CcTrafficLight\bin` 路径。
- Thought Process: hook executable path 必须来自当前安装目录，不能依赖历史默认路径或磁盘固定路径；安装器拥有 `{app}`，应显式传递给部署脚本。
- Options Considered: 每次安装后手动修正 hooks.json；只修改脚本默认值；同时让脚本从自身目录解析并让 installer 传入 `{app}`。
- Chosen Solution: `install-codex-hooks.ps1` 在未传参时优先使用脚本上一级目录的 hook exe，`installer.iss` 的 Run 参数显式传入 `{app}\taskbar_widget_hook.exe`。
- Rationale: 安装到任意用户选择的目录都能写入正确 command，避免旧 `bin` 路径残留。
- Evidence: 隔离 installed-layout fixture 未传 `-HookExecutablePath` 运行成功，7 个 managed command 全部指向 fixture 安装目录；installer.iss 已更新。
