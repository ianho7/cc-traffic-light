- Task: GTV1-B-01 Implement tray icon and base menu actions
- Encountered Problem: The app previously had no system-tray entry point, so adding one required Win32 shell integration without disturbing the existing widget lifecycle.
- Thought Process: I chose to attach tray behavior to the existing main widget window to avoid inventing a second hidden message pump before it was necessary.
- Options Considered: Separate hidden tray window; reuse the widget window as the tray callback target; defer tray until settings exists.
- Chosen Solution: Reuse the widget window as the tray callback target and add a small tray_icon module for icon registration and menu behavior.
- Rationale: That gives a working tray entry point with minimal lifecycle complexity and keeps Phase 2 focused on shell behavior rather than architecture expansion.
