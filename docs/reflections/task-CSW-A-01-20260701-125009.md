- Task: CSW-A-01 cargo check hook binary prerequisites
- Encountered Problem: 需要先确认当前 Rust 工程在切换项目 hooks 前没有基本编译问题，否则后续 hook 配置切换会把运行时错误和构建问题混在一起。
- Thought Process: 先跑成本最低的 `cargo check`，把 P0 的第一步锁定为“当前源码可通过快速校验”。
- Options Considered: 直接 `cargo build`；先 `cargo check` 再 `cargo build`；只做源码阅读不跑命令。
- Chosen Solution: 先运行 `cargo check`，再把更贵的 `cargo build` 放到下一步。
- Rationale: `cargo check` 足够快，能先排除语义和依赖层面的明显问题，为后续 Phase 1 提供最小独立证据。
