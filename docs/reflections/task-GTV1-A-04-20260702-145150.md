- Task: GTV1-A-04 Define settings/detector/widget refresh model
- Encountered Problem: If later phases let each UI surface read files independently, the app will drift into contradictory state views.
- Thought Process: I described shared update channels as runtime signals and introduced a single snapshot entry point to become the future read model.
- Options Considered: Let each surface poll independently; document the rule only; encode signals and snapshot ownership in code now.
- Chosen Solution: Encode runtime signals in runtime_contract and add APP_STATUS_SNAPSHOT as the shared future status entry point.
- Rationale: Even before tray/settings exist, the refresh model now has a clear direction: detectors publish, aggregator consolidates, UI surfaces render.
