- Task: HC-P3-GATE, HC-P3-01 through HC-P3-06
- Encountered Problem: 原 `WM_PAINT` 只绘制固定文本，没有状态输入；但不能把磁盘 IO 放进 paint hot path。
- Thought Process: 现有 Win32 message loop 适合用 `SetTimer` 做 1 秒轮询，状态变化时 invalidate，绘制阶段只消费内存中的 `HookSummary`。
- Options Considered: 文件 watcher；在 paint 时读取状态；timer 轮询并缓存 summary。
- Chosen Solution: 在 `main.rs` 中添加 1000ms timer、全局 `PaintState` 缓存、状态变化日志和基于 state 的文本/背景色绘制。
- Rationale: timer 是当前架构下最小可靠方案；没有修改 taskbar probing、SetParent 或 positioning 逻辑，符合窄范围约束。
