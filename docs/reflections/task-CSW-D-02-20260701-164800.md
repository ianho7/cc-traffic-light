- Task: CSW-D-02 trigger a real Codex state change and verify the widget updates within the polling window
- Encountered Problem: 虽然真实 `active_task_count` 能从 `2 -> 3 -> 4`，但同一实例的 after 采样连续两轮都变成空白，拿不到稳定的可见更新截图。
- Thought Process: 先把问题从“hooks 是否写入”剥离出来，只保留最短链路：fresh widget 启动、抓 before、触发 distinct real thread、2-3 秒内抓 after，并同时用 `taskbar_widget_hook.exe list` 证明状态确实变化。
- Options Considered: 继续相信一次空白图只是偶发；延长等待时间；缩短到 2-3 秒快速采样；把结论收紧为运行期稳定性 blocker。
- Chosen Solution: 用两轮快速采样证明“状态变了，但 after 仍空白”，并把 blocker 明确收敛到 widget 运行期稳定性，而不是继续宣称 redraw pass 或 hook failure。
- Rationale: 这避免把重复的空白 after 误判成状态链路问题，也为下一轮把工作重点切到 widget 生命周期诊断提供了足够证据。
