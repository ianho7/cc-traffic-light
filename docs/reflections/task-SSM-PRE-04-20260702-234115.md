- Task: SSM-PRE-04 确认最小验证命令与 Slint 接入后的新增构建步骤
- Encountered Problem: 需要明确 Slint 引入后是否改变项目的最小验证口径。
- Thought Process: 检查当前 `Cargo.toml` 依赖和仓库约定，判断新增内容主要会落在 `build.rs` 和 `slint-build`，而不是改变验证入口。
- Options Considered: 新增单独验证命令作为前提；保持 `cargo check` / `cargo fmt -- --check` 为最小门槛。
- Chosen Solution: 保持 `cargo check` 与 `cargo fmt -- --check` 为最小验证命令，只在 checklist 中记录后续将增加 Slint 的生成步骤。
- Rationale: 这与仓库现有工作流一致，也避免在基础设施尚未接入前过早扩张验证面。
