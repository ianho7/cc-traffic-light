- Task: HC-P1-GATE, HC-P1-01 through HC-P1-11
- Encountered Problem: 多个短生命周期 hook 进程会并发写同一个 JSON 文件，单纯 rename 不能防止 read-modify-write 丢更新。
- Thought Process: 状态层必须先解决 task key、事件顺序、跨进程锁、损坏恢复和 stale 聚合，否则后续 UI 会被并发任务或旧事件污染。
- Options Considered: 只保存一个全局状态；按 agent 保存状态；保存 task latest snapshot 并聚合 summary。
- Chosen Solution: 新增 `agent_state.rs`，用 `tasks + global_summary + agents.*.summary` schema，`TaskKey = agent_name + "_" + session_id`，Win32 named mutex 保护更新，临时文件 rename 写入。
- Rationale: task-aware schema 可以支持多任务但不扩大 MVP UI；mutex 解决并发写入，event order 解决业务乱序，stale 不参与主 summary 避免长期误报。
