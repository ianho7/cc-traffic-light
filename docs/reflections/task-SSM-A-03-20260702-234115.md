- Task: SSM-A-03 审计正式 settings 项与 AppConfig 映射
- Encountered Problem: 迁移 settings 时不能把未持久化的实验项伪装成正式配置。
- Thought Process: 逐个核对当前 `AppConfig` 字段，区分已经有 schema 的设置和未来还缺持久化落点的设置。
- Options Considered: Slint 页面先放全部想要的项；只有进入 `AppConfig` 的项才能进正式 UI。
- Chosen Solution: 采纳 schema gate，确认当前 general/monitoring/appearance/diagnostics 的既有字段可用，语言配置需要新增 schema。
- Rationale: 这能保证“改了就持久化、重启后保持一致”不是一句空话。
