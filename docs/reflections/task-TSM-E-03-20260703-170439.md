- Task: TSM-E-03 复刻关键交互形态
- Encountered Problem: checklist 要求先把 Nothing 风格的关键交互外形做出来，但不能把 HTML demo 里的演示逻辑或新功能也一起带进来。
- Thought Process: 我把“交互形态”限定为可见结构和控件表现，而不是功能扩展：导航编号、状态卡、InfoRow、ToggleRow、诊断 trace 和胶囊按钮都复刻外观，但仍复用现有保存/刷新入口。
- Options Considered: 只改配色不改结构；完整照搬 HTML demo 的 DOM 和脚本；保留现有逻辑，单独重做关键控件形态。
- Chosen Solution: 在 `App.tsx` 中新增 section、info row、setting row、toggle pill、status badge/dot、trace row 等前端组件模式，并把 overview/general/monitoring/appearance/diagnostics/about 全部套入新的交互外形。
- Rationale: 这样既完成了关键视觉与交互壳层的迁移，又没有新增配置项、模拟动作或额外状态来源，风险最低。
