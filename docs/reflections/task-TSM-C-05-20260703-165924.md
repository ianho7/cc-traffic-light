- Task: TSM-C-05 宿主管理单一 settings 进程生命周期
- Encountered Problem: 仅有 IPC server 和 Tauri scaffold 还不能避免 tray 重复点击时拉起多个 settings 实例，而且主进程退出后也需要明确回收策略。
- Thought Process: 我把这一步限定在宿主生命周期层，不提前切默认入口；先让 `taskbar-widget` 自己持有唯一 child 句柄、复用仍存活的进程、在窗口销毁时显式回收，再保留 Slint fallback 作为未切默认前的保守路径。
- Options Considered: 继续依赖 Tauri 单实例能力自行约束；每次点击都无状态地重新 spawn；由宿主维护单一 child 并在可见窗口存在时优先 focus。
- Chosen Solution: 新增 `settings_process.rs`，用 `OnceLock<Mutex<...>>` 在宿主侧跟踪单一 Tauri settings child，点击 tray 时优先复用并尝试聚焦已有窗口，仅在进程已退出时重新拉起；`WM_DESTROY` 时统一 shutdown。
- Rationale: 单实例约束放在 Win32 host 侧更符合当前迁移边界，能直接满足 checklist 对“避免多实例”的要求，同时不把默认入口切换和生命周期治理混在同一轮里，回退风险更低。
