- Task: CSW-D-04 check state and verifier evidence before blaming UI when widget does not visibly update
- Encountered Problem: `after.png` 为空白时，很容易直接把问题归因到“真实事件没有驱动 redraw”。
- Thought Process: 先核对同轮的真实状态 verifier，再检查 widget 进程本身是否还活着，区分“状态没变”和“UI 进程已退出”。
- Options Considered: 直接判定 redraw 失败；只看像素 delta；补查 list 输出、pid 文件和进程状态。
- Chosen Solution: 用 `taskbar_widget_hook.exe list`、`widget.pid.txt` 和进程检查把结论收紧为“这轮截图更像进程退出”。
- Rationale: 这样不会把一次运行时退出误写成状态链路失败。
