- Task: SSM-A-01 拆分 settings_window.rs 职责
- Encountered Problem: 当前 `settings_window.rs` 把显示、状态、命令和持久化揉在一起，若不先拆认知边界，迁移时容易复制耦合。
- Thought Process: 按 UI 迁移常见分层，把现有逻辑拆成展示层、状态适配层、命令层、持久化契约层和本地化契约层。
- Options Considered: 保持现有单文件心智；先做职责拆分，再设计 Slint host。
- Chosen Solution: 形成五层职责划分，并把“本地化契约层当前缺失”列为显式缺口。
- Rationale: 这样后续可以只把展示层和一部分适配层迁到 Slint，而不重写整个运行时。
