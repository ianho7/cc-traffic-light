- Task: AUD-6-08 持久化通知去重
- Encountered Problem: 原启动通知只用进程内 AtomicBool 去重，应用重启后同一 hook 状态会重复提示。
- Thought Process: 去重键应由两个 Agent 的状态组成；当两者恢复 Active 时清除旧键，避免未来状态变化被永久抑制。
- Options Considered: 仅扩大进程内缓存；写独立 marker 文件；将 fingerprint 放入已有 diagnostics 配置。
- Chosen Solution: 在 DiagnosticsConfig 增加可选 `last_hook_notification_key`，启动时比较并持久化 `codex:<status>;claude:<status>`，全部 Active 时清除。
- Rationale: 复用现有原子配置持久化和备份路径，不增加新的状态文件或隐含用户数据目录。
- Evidence: `app_config.rs`、`settings_bridge.rs`、`main.rs`；shared-core 配置测试、7 个 settings bridge 测试和 TypeScript 检查通过。
- Status: completed
