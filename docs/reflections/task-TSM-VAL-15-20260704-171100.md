- Task: TSM-VAL-15 Tauri 设置页概览中的 Codex 状态必须与宿主实时检测快照一致
- Encountered Problem: `%APPDATA%\CcTrafficLight\state.json` 和进程探测都表明 Codex 正在工作，但 Tauri 设置页仍显示 `undiscovered / unknown / updated_at=0`。
- Thought Process: 我先把问题拆成三层对照：磁盘状态、detector 构造逻辑、named-pipe IPC 返回值。这样可以确认是业务判断错了，还是宿主缓存/转发链路错了。
- Options Considered: 1. 直接改前端渲染兜底。2. 修改 detector 判定优先级。3. 验证宿主缓存初始化和同步链路是否把空快照放大。
- Chosen Solution: 修正 `sync_settings_hosts()`，让 settings bridge 和 Tauri IPC 统一吃 `APP_STATUS_SNAPSHOT` 的实时值；再新增 `validate-tauri-settings-snapshot-alignment.ps1`，用 state file 对齐 named-pipe 快照。
- Rationale: 前端只是快照消费者，真正的问题是宿主把 `AppStatusSnapshot::empty()` 初始化进 bridge 后，又从同一缓存自我回读，导致空值永久存在。把同步源收敛到主循环实时快照，才是正确修复点。
