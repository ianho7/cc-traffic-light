- Task: TSM-H-05 验证 Tauri 异常退出时，宿主侧有清晰可诊断日志，并允许再次打开
- Encountered Problem: 在 settings 进程被强杀后，早期脚本只知道“没找到 settings 窗口”，但不能区分是宿主没收到命令、spawn 失败，还是 settings 退出后未能重拉起。
- Thought Process: 先让 runtime log 明确记录 `settings-process` 的 `spawned`、`reused`、`previous ... exited` 和 `terminated ...` 事件，再用脚本主动强杀 settings 并重新触发一次 Open Settings，验证宿主是否能发现旧进程已退出并再次拉起。
- Options Considered: 只看进程列表；只看窗口；结合 runtime log 与脚本断言同时验证异常退出后的重新拉起。
- Chosen Solution: 保留 `Stop-Process -Id $report.second_settings_pid -Force` 的 settings 异常退出步骤，再检查 `reopen_after_forced_kill_spawned = true`，同时核对 runtime log 中的 `previous tauri settings exited status=...` 与后续新的 `spawned tauri settings pid=...`。
- Rationale: 这样既验证了“异常退出后能恢复”，也验证了宿主日志具备足够诊断性。最新 runtime log 和 report 都满足这两个条件。
