- Task: GTV1-C-05 Wire detector output into shared status aggregation and UI refresh
- Encountered Problem: The UI snapshot still came from direct state-file mapping, so adding a detector module would have had no practical effect unless startup and timer refresh paths started consuming it.
- Thought Process: I replaced the direct AppStatusSnapshot construction in main.rs with detector-driven construction during both initialization and timer polling.
- Options Considered: Keep detector unused until more sources exist; expose detector only to settings; make detector the single snapshot producer immediately.
- Chosen Solution: Make detector the single snapshot producer immediately.
- Rationale: That ensures every later source addition automatically updates settings, tray refresh, and any future tooltip rendering through one shared path.
