- Task: TSM-PRE-06 确认归档策略
- Encountered Problem: 旧 Slint settings 代码后续一定会退场，但如果没有先定归档策略，很容易在切入口时直接删掉参考实现。
- Thought Process: 我沿用 checklist 里的要求，把归档视为阶段性收口动作，而不是最终清理时的随意删除。
- Options Considered: 迁移完成后直接删除；长期双轨并存；先保留，最终迁入 `archive/`。
- Chosen Solution: 明确旧 Slint settings 在最终阶段迁入 `archive/`，不是直接删除。
- Rationale: 这能保留迁移依据和 fallback 参考，同时避免长期双主入口。
