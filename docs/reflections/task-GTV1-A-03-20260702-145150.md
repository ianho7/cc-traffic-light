- Task: GTV1-A-03 Define unified aggregate status structure
- Encountered Problem: Existing hook summaries only describe the old widget state, while V1 needs source-level status, confidence, detection method, and widget mount state.
- Thought Process: I designed a UI-facing snapshot that can represent both sources consistently and carry enough metadata for settings, tray, diagnostics, and widget rendering.
- Options Considered: Keep using HookSummary directly; add fields ad hoc to main.rs; define a dedicated UI/runtime snapshot model.
- Chosen Solution: Define a dedicated ui_state module with source status and app snapshot types.
- Rationale: The product-level state model is broader than the current hook schema and needs to stay stable as detectors evolve.
