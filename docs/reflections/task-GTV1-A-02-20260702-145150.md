- Task: GTV1-A-02 Define user-level config schema
- Encountered Problem: The settings surface was already specified, but there was no durable schema for autostart, tray behavior, monitoring toggles, or appearance settings.
- Thought Process: I translated the agreed settings scope into a serializable V1 config with defaults, load behavior, and a stable path.
- Options Considered: Delay config until settings UI exists; reuse state.json; create a dedicated config.json schema.
- Chosen Solution: Create a dedicated app_config module and config.json schema.
- Rationale: Runtime settings and observed state have different lifecycles; mixing them would make detector and UI work harder later.
