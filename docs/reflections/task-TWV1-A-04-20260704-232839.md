- Task: TWV1-A-04 Decide the window style and layered combination for phase 1
- Encountered Problem: The existing layered mode only covered whole-window opacity, not per-pixel transparency.
- Thought Process: I preserved the current taskbar parenting model and changed only the rendering submission path needed for transparent output.
- Options Considered: Stay on opaque layered attributes; remove layering entirely; keep popup-parented hosting and add a per-pixel layered mode.
- Chosen Solution: Keep `StyleMode::PopupParented`, add `LayeredMode::PerPixel`, and submit ARGB frames with `UpdateLayeredWindow()`.
- Rationale: This keeps the host model stable and isolates the visual change to the rendering path instead of widening scope into attach logic.
