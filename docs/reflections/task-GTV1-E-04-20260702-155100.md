- Task: GTV1-E-04 实现 widget 挂载失败后的托盘降级逻辑
- Encountered Problem: 之前无论 attach / layout 是否成功，主 widget 窗口都会继续按已挂载状态运行，这会让 UI 误报“已附着”，也不符合“失败时至少保留 tray”的产品要求。
- Thought Process: 为了不碰 `taskbar.rs` 的稳定 attach 实现，本轮把降级判断放在主循环层，只读取 attach / layout 结果并决定 widget 是否显示、当前 mount state 是什么。
- Options Considered: 1. 在 `taskbar.rs` 内部直接做隐藏逻辑。2. 让 widget 始终显示，只在 settings 里提示失败。3. 在 `main.rs` 增加独立运行时状态层，根据 attach / layout 结果切换 `Attached` / `TrayOnly`。
- Chosen Solution: 采用主循环层状态机；初始 attach 或 layout 失败时，把 widget 标记为 `TrayOnly` 并隐藏窗口，同时保留 tray / settings / detector 正常运行。
- Rationale: 这样不会破坏现有稳定 attach 路径，降级策略也保持在更高层，后续要扩展重试或诊断信息时都更容易控制。
