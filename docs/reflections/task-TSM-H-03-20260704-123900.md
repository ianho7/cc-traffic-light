- Task: TSM-H-03 验证重复打开、关闭、隐藏、再打开 settings 时，不会留下孤儿进程或僵尸窗口
- Encountered Problem: 早期验证被 `TaskDialogIndirect` loader 错误和脚本窗口定位过于依赖固定标题同时阻断，导致“单进程复用”和“关闭后重开”都无法可靠判断。
- Thought Process: 先把 loader 问题缩小到 workspace feature 污染，再让 lifecycle 脚本分包构建宿主与 settings，随后补上按 PID fallback 查找 Tauri 窗口和等待窗口出现，确保重复打开/关闭走的是同一条真实 Tauri 生命周期路径。
- Options Considered: 继续人工点击验证；只看进程数不看窗口；补强自动脚本，验证复用、关闭退出和关闭后重开三段链路。
- Chosen Solution: 使用修正后的 [validate-tauri-settings-lifecycle.ps1](/D:/project/cc-traffic-light/taskbar-widget/scripts/validate-tauri-settings-lifecycle.ps1) 自动验证 `open_spawned_once`、`reopen_reused_same_process`、`close_exited_process`、`reopen_after_close_spawned` 四个断言。
- Rationale: 这比人工观察更稳定，也直接对应“无孤儿进程、无僵尸窗口、关闭后能再打开”的目标。最新 report 中相关布尔项均为 `true`。
