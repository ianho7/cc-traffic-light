- Task: TSM-F-01 让 Tauri UI 读取真实配置与真实状态
- Encountered Problem: 仅靠“页面看起来像 settings”并不能证明 Tauri 已经接到真实宿主数据，尤其在 fake backend 仍保留开发兜底的情况下，很容易把演示通路误判成真实集成。
- Thought Process: 我把验证拆成两层：前端层确认 `App.tsx` 确实消费 `bootstrap/getSnapshot/getSettings` 返回值，宿主层再用 live named-pipe 请求证明 `get_snapshot` 和 `get_settings` 来自真实 host bridge。
- Options Considered: 1. 只读代码。2. 只看运行截图。3. 代码覆盖审计加 live named-pipe 读模型验证。
- Chosen Solution: 新增 `verify-settings-coverage.ps1` 做静态绑定审计，并用 `validate-tauri-settings-read-model.ps1` 在真实 host 上验证 live snapshot/settings 读取。
- Rationale: 这比单纯截图或口头判断更可复用，也更适合作为 checklist 的持续证据。
