- Task: TSM-C-04 落地保守本地 IPC 路径
- Encountered Problem: 仅有协议文档和 fake backend 还不足以证明迁移边界真正成立，宿主与新 settings 之间仍然没有真实传输层。
- Thought Process: 我保持配置文件为事实源不变，只把命令和即时生效控制通过 named pipe 传递；同时让 Tauri backend 在连接宿主失败时仍能回退到 fake mode，避免把当前 scaffold 再打断。
- Options Considered: 继续停留在 fake backend；改用 TCP/HTTP 本地服务；落地 Windows named pipe server/client，并保持 config file 为事实源。
- Chosen Solution: 在 `taskbar-widget` 中新增 named pipe server，在 `taskbar-settings-tauri` 中新增 named pipe client，二者都复用 shared-core 的协议类型。
- Rationale: 这满足了“本地 IPC 负责命令与即时生效，配置文件仍作为持久化事实源”的迁移要求，同时不需要把 WebView 直接暴露给宿主内部实现。
