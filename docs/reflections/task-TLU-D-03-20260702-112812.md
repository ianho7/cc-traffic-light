- Task: TLU-D-03 验证真实事件到 UI 更新的轮询延迟结论
- Encountered Problem: 真实 hook 主链路修正前，状态语义不可信，无法把任何延迟观察当成有效证据。
- Thought Process: 先修正 `Stop` / 失败路径映射，再用成功、失败、waiting 三类真实对话确认 widget 都能自动更新到明确终态。
- Options Considered: 在错误映射未修正时就记录延迟；先修映射再收口这一验证项。
- Chosen Solution: 先修映射，再以真实自动状态切换通过作为轮询链路可用证据。
- Rationale: 只有终态语义正确，延迟观察才有意义。
