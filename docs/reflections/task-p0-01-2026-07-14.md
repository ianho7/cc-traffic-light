# P0-01 Reflection

- Task: 定义最小 MaterialGroup 配置模型。
- Encountered Problem: 素材图片不能内嵌在 JSON，否则配置会膨胀且 host 无法高效缓存。
- Thought Process: 对照现有配置边界和 PNG 渲染路径，只保留资源身份与三个本地文件路径。
- Options Considered: Base64 配置；通用资产表；固定五字段模型。
- Chosen Solution: `MaterialGroup { id, name, green_path, yellow_path, red_path }`。
- Rationale: 直接覆盖 MVP 的固定三槽位，未引入资源平台抽象。
