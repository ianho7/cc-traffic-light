- Task: GTV1-E-03 实现只读诊断页：来源类型、更新时间、可信度、错误摘要、手动刷新
- Encountered Problem: 现有 settings 只有 overview 和 general，缺少一个能解释 detector 结论的只读面板；同时 tray 菜单中的 `Refresh Detection` 也还没有触发真实探测，诊断页如果只显示静态字段价值很低。
- Thought Process: 先把 diagnostics 定位为“解释当前统一快照”的只读页，而不是第二套交互中心。这样 UI 只消费 `AppStatusSnapshot`，手动刷新则复用主窗口已有刷新链路，避免 settings 直接探测底层来源。
- Options Considered: 1. 只放文本占位。2. 让 diagnostics 自己直接读取来源文件。3. 让 diagnostics 只读展示统一快照，并通过主窗口消息触发统一 refresh。
- Chosen Solution: 增加 `DIAGNOSTICS` 页面，显示 widget 挂载状态、最近刷新时间、错误摘要，以及 `Codex` / `Claude` 的状态、来源方法、可信度、更新时间和 message；`REFRESH NOW` 与 tray 刷新都统一走主窗口刷新命令。
- Rationale: 这样保持了 runtime 分层，settings 不直接碰 detector 实现细节，同时用户终于能看到“为什么是这个状态”的证据面板；刷新语义也在 tray 和 diagnostics 之间统一了。
