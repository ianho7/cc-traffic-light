- Task: PRE-02 确认将使用一个新的独立 Rust 二进制项目，而不是直接改造现有大工程
- Encountered Problem: 当前仓库只有文档，没有现成代码基线，直接改造“主工程”实际上没有着力点。
- Thought Process: 对照 checklist 的 MVP 边界，优先满足“独立可执行程序”和“最小闭环”，避免为将来不存在的集成点提前设计。
- Options Considered: 直接在仓库根堆文件；等待未来主工程出现后再建；立刻创建独立 Rust bin 项目。
- Chosen Solution: 在仓库根下创建 `taskbar-widget` 独立项目。
- Rationale: 这条路径最符合 MVP 目标，也最容易把普通窗口基线和后续任务栏嵌入隔离验证。
