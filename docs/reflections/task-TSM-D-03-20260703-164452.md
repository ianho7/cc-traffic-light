- Task: TSM-D-03 建立 `src-tauri/` 最小命令通道并用假数据打通
- Encountered Problem: 若直接等待真实宿主 IPC，Tauri backend 和前端之间将没有任何可验证的闭环。
- Thought Process: 我让 `src-tauri` 先依赖 shared-core，并实现一个内存态 fake backend，暴露与最终协议一致的 Tauri commands。
- Options Considered: 只做静态前端；直接实现 named pipe client；先用 fake state 打通 `bootstrap_window`、读配置、读状态、保存设置、请求刷新、通知应用。
- Chosen Solution: 在 `src-tauri/src/lib.rs` 中实现 fake backend，并通过 `tauri::generate_handler!` 注册最小命令通路。
- Rationale: 这让窗口创建、前端 invoke 和协议 DTO 三层先形成闭环，后续替换 transport 时风险更低。
