- Task: SSM-E-02 把 Diagnostics 的 Refresh Now 接到现有主窗口刷新命令
- Encountered Problem: 诊断页按钮如果只更新 Slint 本地状态而不触发主窗口刷新，就只是假的交互。
- Thought Process: 继续沿用现有 Win32 主窗口 `WM_COMMAND + TRAY_CMD_REFRESH` 命令路径，让 Slint 只是触发器。
- Options Considered: Slint 线程直接重跑 detector；继续复用主窗口刷新命令。
- Chosen Solution: `request_refresh` callback 调用共享 backend，再由 backend 向主窗口投递现有刷新命令。
- Rationale: 这保证 detector 主路径不分叉，也符合“不破坏现有 widget/detector 主路径”的约束。
