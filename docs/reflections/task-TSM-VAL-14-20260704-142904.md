- Task: TSM-VAL-14 旧 Slint 代码已归档或明确降级为 fallback，不再形成真实双主入口
- Encountered Problem: 仅凭 tray 默认打开 Tauri 还不足以证明没有双主入口，因为宿主仍可能在运行时初始化旧 UI、同步状态或托管旧事件循环。
- Thought Process: 我用“是否参与活跃构建、是否参与运行时初始化、是否仍能被默认入口触达”三个问题来判断 Slint 是否真的退场。
- Options Considered: 1. 只看 tray 行为。2. 只看 Cargo 依赖。3. 以构建、运行时和 fallback 路径三层一起验证。
- Chosen Solution: 让 Slint 同时退出这三层，只保留 Win32 fallback，并用 lifecycle 验证再次确认默认 settings 路径仍正常。
- Rationale: 这才符合“不再形成真实双主入口”的定义，而不是仅仅换了一个默认按钮分支。
