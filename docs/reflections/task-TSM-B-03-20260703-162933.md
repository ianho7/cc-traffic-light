- Task: TSM-B-03 在 shared-core 中建立 settings service 边界
- Encountered Problem: 如果 shared-core 只有数据结构，没有 service 边界，后续 Tauri 仍会被迫直接调用宿主内部实现细节。
- Thought Process: 我在 shared-core 里先定义最小服务契约，让“读配置、写配置、读状态、应用变更”有统一接口，再由宿主 bridge 去实现平台侧逻辑。
- Options Considered: 只迁 DTO；在宿主里直接暴露一堆自由函数；在 shared-core 中定义 `SettingsService` trait 和错误类型。
- Chosen Solution: 在 `shared-core::settings_service` 中定义 `SettingsService`、`SettingsServiceError`，并包含 `load_settings`、`save_settings`、`read_status_snapshot`、`apply_settings_change`。
- Rationale: 这给 Tauri IPC 和宿主 bridge 提供了明确的协议面，而不是继续复用 UI 文件里的 helper。
