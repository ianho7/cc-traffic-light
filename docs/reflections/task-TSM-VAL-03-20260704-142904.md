- Task: TSM-VAL-03 pnpm build 通过
- Encountered Problem: 前端构建最初不是代码失败，而是连续卡在 PNPM supply-chain 配置、`esbuild` postinstall 权限、无 TTY modules purge，以及 `tsc`/`vite` Volta wrapper 四层环境问题上。
- Thought Process: 我先把 `pnpm-workspace.yaml` 的 `allowBuilds.esbuild` 从占位值修正为布尔值，再做一次无沙箱 `pnpm install` 恢复依赖，最后把构建脚本改为 PowerShell 直接用 `npm_node_execpath` 驱动 `tsc.js` 和 `vite.js`。
- Options Considered: 1. 只记录环境问题不解。2. 用本机绝对 Node 路径硬编码到 package script。3. 修正 PNPM 配置并把 build 入口做成仓库内可重复脚本。
- Chosen Solution: 保持根 `pnpm build` 入口不变，但让它落到 `taskbar-settings-tauri/scripts/build.ps1`，再用 `CI=true pnpm build` 完成无 TTY 验证。
- Rationale: 这样既完成了 checklist 验证项，也把未来同类环境问题压缩成仓库内的稳定入口，而不是当前机器专用技巧。
