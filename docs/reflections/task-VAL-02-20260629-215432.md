- Task: VAL-02 `cargo run` 后普通窗口基线可工作，作为 Phase 1 独立验收信号
- Encountered Problem: `cargo run` 的运行证据既需要进程层面的启动/退出，也需要窗口层面的实际显示，不适合只看一类信号。
- Thought Process: 先检查自动化运行结果，再把用户的手动确认加入验证链，避免把“进程没崩”误当成“窗口基线可工作”。
- Options Considered: 只依赖编译通过；只依赖进程日志；结合日志、窗口探测和人工确认。
- Chosen Solution: 使用 `cargo check`、运行日志、两次窗口探测和用户确认共同支撑 `VAL-02`。
- Rationale: 这比单纯的构建结果更能证明 Phase 1 真正成立。
