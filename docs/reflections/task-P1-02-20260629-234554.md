# Task Reflection

- Task: `P1-02` 保留 `Start`、`TrayNotifyWnd` 作为定位锚点输入，停止把 `task_switch` / `rebar` 作为 Win11 主线父窗口候选
- Encountered Problem: 旧代码把 `task_switch` 继续放在 `TrayNotifyWnd` 失效时的默认回退链里，Win11 主线宿主和调试候选混在一起。
- Thought Process: 按 TrafficMonitor 的 Win11 路线收紧职责，把 `task_switch` 降回显式调试选项，不再进入默认锚点链。
- Options Considered: 完全删除 `task_switch` 支持；保留为 env 调试选项但退出默认主线。
- Chosen Solution: 默认锚点链改成 `TrayNotifyWnd -> Start -> Shell_TrayWnd`，`task_switch` 只在显式指定时参与。
- Rationale: 这既保留诊断能力，又不让普通运行继续偏离 Win11 主线。
