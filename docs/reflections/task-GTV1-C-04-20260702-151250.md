- Task: GTV1-C-04 Make process detection a fallback-only source that never claims high-confidence working state
- Encountered Problem: The detector contract reserved a process-level fallback, but leaving it unimplemented meant the zero-config path still lacked any non-state-file fallback signal.
- Thought Process: I added a conservative ToolHelp-based process scan for Codex and Claude-related executable names, but mapped presence only to degraded idle instead of any active working state.
- Options Considered: Skip process fallback entirely; mark process presence as working; mark process presence as a degraded idle fallback only.
- Chosen Solution: Mark process presence as a degraded idle fallback only.
- Rationale: That satisfies the requirement that process detection is allowed as a fallback while preserving the rule that it must not by itself imply high-confidence active work.
