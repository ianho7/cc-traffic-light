- Task: PRE-03 确认项目最小结构：`Cargo.toml`、`src/main.rs`、`src/taskbar.rs`、`src/win32.rs`
- Encountered Problem: Phase 1 只需要普通窗口基线，但后面 Phase 2 到 Phase 4 又需要最小状态和 Win32 辅助，单文件很快会变脏。
- Thought Process: 保持最小文件数，同时为后续 `SetParent` 和句柄探测预留最基础的放置点。
- Options Considered: 全部逻辑塞进 `main.rs`；一次拆成很多模块；按文档建议保留 4 个最小文件。
- Chosen Solution: 使用 `Cargo.toml`、`src/main.rs`、`src/taskbar.rs`、`src/win32.rs` 四文件结构。
- Rationale: 这既满足当前最小复杂度，又不给后续阶段制造重构负担。
