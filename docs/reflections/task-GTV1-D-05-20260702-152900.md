- Task: GTV1-D-05 把总聚合状态同步到极简 tray 图标
- Encountered Problem: 现有 tray 只有默认应用图标和 tooltip，无法把总聚合状态直接映射到托盘视觉，同时 Rust `OnceLock` 不能直接缓存 `HICON` 句柄类型。
- Thought Process: 先确认产品边界是“极简聚合图标”而不是把 widget 缩小搬进 tray，因此选择单灯语义。实现上比较了继续使用默认图标、引入资源文件和运行时生成图标三种路径，最终优先选择运行时生成，避免新增资源管理链路。
- Options Considered: 1. 保持默认图标只更新 tooltip。2. 新增多套 `.ico` 资源并在运行时切换。3. 运行时生成 16x16 ARGB 单灯图标并按 overall state 切换。
- Chosen Solution: 采用运行时生成的极简单灯图标，并把 `overall_state` 映射为绿 / 黄 / 红 / 深灰 / 浅灰五种托盘显示；图标句柄缓存改为 `isize` 存储，再在使用点转回 `HICON`。
- Rationale: 这样能满足 V1 “tray 只展示总聚合状态”的目标，不需要引入资源文件或第二套 UI 资产流程，同时保留纯 Win32 + Rust 的边界，编译验证也最直接。
