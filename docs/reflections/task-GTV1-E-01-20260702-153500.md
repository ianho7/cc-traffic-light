- Task: GTV1-E-01 实现用户级配置持久化与加载
- Encountered Problem: GUI 和 tray 已经开始依赖统一配置，但如果没有稳定的用户级保存位置，设置窗口切页、基础开关和后续 diagnostics 都无法形成闭环。
- Thought Process: 先确认 V1 只需要当前用户级本地配置，不需要安装器、系统级目录或数据库。然后把配置 schema、默认值、磁盘路径、加载诊断和保存接口统一收进一个独立模块，避免 UI 代码直接拼路径或处理 JSON。
- Options Considered: 1. 把配置散落在 `main.rs` / `settings_window.rs`。2. 使用注册表保存全部配置。3. 使用 `%APPDATA%\\CcTrafficLight\\config.json` 保存结构化配置，并在模块层提供 load/save 诊断。
- Chosen Solution: 采用 `%APPDATA%\\CcTrafficLight\\config.json` 作为 V1 用户级配置存储，封装在 `app_config.rs` 中，并由 settings 交互直接复用该模块。
- Rationale: JSON 文件最容易诊断、最容易扩展，也不会把非启动项设置错误地耦合到注册表；对纯 Win32 + Rust 的 MVP 来说，这是最稳妥的持久化边界。
