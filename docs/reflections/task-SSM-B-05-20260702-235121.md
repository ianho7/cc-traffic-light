- Task: SSM-B-05 约束 Slint 侧不直接硬编码业务字符串
- Encountered Problem: 如果把业务文案直接塞进 `.slint`，后续统一 i18n 层时会重新拆一遍组件文本。
- Thought Process: 区分“组件结构”和“字符串来源”，让 `.slint` 只声明属性，不直接承担业务文案定义。
- Options Considered: 在 `.slint` 中直接写默认文案；让 Rust host 为所有可见文本赋值。
- Chosen Solution: 组件只保留字符串属性，当前由 Rust host 注入过渡文案，后续再接真实 i18n 层。
- Rationale: 这样先满足 Phase 2 的静态可视化目标，同时保留后续统一本地化的可演进性。
