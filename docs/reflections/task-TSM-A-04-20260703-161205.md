- Task: TSM-A-04 确定 shared-core 第一批内容
- Encountered Problem: 如果 shared-core 边界切得太大，会把 Win32 或 Tauri runtime 细节带进去；切得太小，则不能承接迁移目标。
- Thought Process: 我只选择纯 Rust 业务类型和 service 门面进入 shared-core，同时明确哪些平台细节必须留在宿主侧。
- Options Considered: 全量迁 `settings_window` backend；只迁 DTO；迁 DTO + settings service + 错误类型。
- Chosen Solution: shared-core 第一批承接 `AppConfig`、`AppStatusSnapshot` 投影、settings service、DTO 和错误类型。
- Rationale: 这既能服务 Tauri IPC 契约，又不会污染共享层。
