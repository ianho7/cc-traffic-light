- Task: TSM-F-02 接入 1 秒轮询刷新策略
- Encountered Problem: 现有 Tauri UI 只在 bootstrap 时读取一次 snapshot/settings，Overview 和 Diagnostics 会很快过时，无法承担 read-only 观察面的职责。
- Thought Process: 我把这一轮严格限制在前端读路径：复用已有 `get_snapshot` / `get_settings` Tauri commands，每秒静默拉取一次 read model，并避免并发轮询叠加或把瞬时失败变成前端噪音。
- Options Considered: 继续只保留手动刷新；在宿主侧新增推送通道；先用 1 秒轮询持续拉取 snapshot/settings。
- Chosen Solution: 在 `taskbar-settings-tauri/src/App.tsx` 中新增 1 秒 `setInterval` 轮询，使用 `Promise.all([getSnapshot(), getSettings()])` 刷新当前 bootstrap read model，并用 `pollInFlightRef` 避免重叠请求。
- Rationale: 这满足了 F-02 对 detector 状态与运行摘要轮询刷新的要求，同时不扩大 IPC 复杂度，也不打断现有保存和 fallback 路径。
