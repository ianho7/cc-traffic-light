- Encountered Problem: 之前的结论混在一起：既怀疑真实 hooks，也怀疑 widget 生命周期，还受单点 after 截图假阴性影响，没有一个稳定的 pass/fail loop。

- Options Considered: 继续用零散人工截图；只加日志不做 harness；同时补日志和统一脚本反馈回路。
- Chosen Solution: 增加 `TASKBAR_MVP_RUNTIME_LOG_FILE`、`[DEBUG-LIVE-01]` 日志，以及 `diagnose-widget-liveness.ps1`，并实际跑通 baseline / fixture replay。
- Rationale: 这让后续诊断从“猜原因”变成“跑 loop 看哪一层 first fail”，而且已经实证排除了两条高概率误判方向。
