- Task: SSM-B-04 建立 i18n 资源骨架
- Encountered Problem: 用户已经要求中英国际化覆盖全局用户可见字符串，必须避免 Slint 先走单语硬编码。
- Thought Process: 先建立 `en.json` 和 `zh-CN.json` 的 key 骨架，为下一阶段统一 i18n 层做准备。
- Options Considered: 先把文案直接写在组件里；先建资源文件骨架再逐步接线。
- Chosen Solution: 创建中英两份 JSON 资源，覆盖当前静态 settings 结构的主要 key。
- Rationale: 这能让后续从“静态字符串”切到“统一资源访问层”时不再返工命名和覆盖范围。
