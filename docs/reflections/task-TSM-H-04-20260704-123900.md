- Task: TSM-H-04 验证主进程退出时能正确回收或通知 Tauri settings 进程
- Encountered Problem: 初版脚本用 `Stop-Process -Force` 强杀宿主，强杀不会执行 Rust 的 `WM_DESTROY` 清理路径，导致“宿主退出后 settings 未回收”这个失败并不代表产品逻辑真的有问题。
- Thought Process: 区分“异常强杀宿主”和“正常关闭宿主”两种语义后，把最后一步改成向宿主窗口发送 `WM_CLOSE`，让 `main.rs` 中 `WM_DESTROY -> shutdown_managed_tauri_settings()` 的真实收尾逻辑执行。
- Options Considered: 保留强杀断言；手工退出宿主；在脚本中改为 `WM_CLOSE` 正常关闭后再验证 settings 是否退出。
- Chosen Solution: 给 lifecycle 脚本新增 `Send-HostClose`，并等待 `host process to exit after WM_CLOSE` 与 `settings process to exit after host shutdown` 两段断言通过。
- Rationale: 这样验证的是产品真实的正常退出路径，而不是操作系统级强杀语义。最新 report 中 `host_exit_code = 0` 且 `host_shutdown_reaped_settings = true`。
