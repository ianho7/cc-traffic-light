- Task: TSM-C-01 让 `settings_slint.rs` 改为调用 shared-core / 统一 settings service
- Encountered Problem: 当前 Slint settings callback 仍然直连 `settings_window.rs`，这会把 fallback UI 文件继续当成正式 backend。
- Thought Process: 我没有直接重写 Slint UI，而是先把 callback 写路径改成统一 bridge，使当前主入口开始脱离 fallback UI 的内部状态。
- Options Considered: 保持 Slint 继续依赖 `settings_window.rs`；一步切到 Tauri；先让 Slint 回调改走统一 bridge。
- Chosen Solution: 将 `settings_slint.rs` 的配置写回、语言切换、刷新命令和 snapshot 读取改为调用 `settings_bridge.rs`。
- Rationale: 这一步先切断最关键的业务依赖，为后续替换 presentation host 创造条件。
