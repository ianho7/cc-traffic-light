- Task: SSM-B-01 接入 Slint 依赖和构建脚本
- Encountered Problem: 需要在不干扰现有 Win32 主路径的前提下，把 Slint 编译链先接进当前 Rust 项目。
- Thought Process: 先只增加 `slint`、`slint-build` 和 `build.rs`，让 UI 文件具备独立编译能力，不改默认 settings 入口。
- Options Considered: 先直接替换主入口；先只接入编译基础设施。
- Chosen Solution: 在 `Cargo.toml` 增加 Slint 依赖，并通过 `build.rs` 编译 `ui/settings.slint`。
- Rationale: 这样能先验证工程可集成，再逐步接入运行时桥接，风险最低。
