- Task: TSM-CLN-01 删除不再使用的 Slint-only 资源引用和构建残留
- Encountered Problem: 源码里的 `settings_slint` 模块、`slint` 依赖、`build.rs` 编译步骤和旧文件树说明都属于典型的 Slint-only 残留。
- Thought Process: 清理必须覆盖三层：运行时代码引用、构建图依赖、仓库结构说明，否则仍会在维护时造成假阳性。
- Options Considered: 1. 只删代码调用。2. 只删 Cargo 依赖。3. 连同 FILETREE、归档说明和 checklist 一起收口。
- Chosen Solution: 同时删除活跃引用、精简构建配置、移动旧资源到 `archive/`，并更新 FILETREE 与 checklist。
- Rationale: 这样清理结果才是可审计的，不会出现“代码没用但文档还说在用”或“文档改了但构建还在编译”的残留状态。
