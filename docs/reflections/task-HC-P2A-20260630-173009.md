- Task: HC-P2A-GATE, HC-P2A-01 through HC-P2A-05
- Encountered Problem: 真实 hook 未接入时仍需要构造 summary、清理卡住 task，并验证 widget 轮询路径。
- Thought Process: debug 能力如果绕过状态层，会导致验证结果不可信；它必须走同一 mutex、schema、summary 和原子写入逻辑。
- Options Considered: 只用人工 JSON hook；添加独立调试状态文件；在同一 hook binary 中实现 `set/clear/list`。
- Chosen Solution: 在 `taskbar_widget_hook.exe` 中实现 `set <task_key> <state>`、`clear <task_key>`、`list`。
- Rationale: 单 binary 更易配置，debug 命令复用同一状态路径，后续 widget 验证和卡住 task 清理都有确定入口。
