- Task: Realign the active project Codex hook command path with the workspace-root Cargo target directory
- Encountered Problem: The running host and the rebuilt `taskbar_widget_hook.exe` under the workspace root had the new strict `Stop -> done` semantics, but the active project `.codex/hooks.json` still pointed Codex lifecycle hooks at `taskbar-widget\target\debug\taskbar_widget_hook.exe`, an older binary from 2026-07-03.
- Thought Process: I compared the timestamp of the binary currently referenced by `.codex/hooks.json` against the freshly rebuilt workspace-root binary. The active hook path was stale, which explained why the user could restart `.\target\debug\taskbar-widget.exe` and still get old state-writing behavior.
- Options Considered: Keep the old path and try to rebuild the per-crate target; rely on documentation only; switch the active hook config and examples to the workspace-root `target\debug` location that Cargo actually uses for this workspace.
- Chosen Solution: Update `.codex/hooks.json` to call `D:\project\cc-traffic-light\target\debug\taskbar_widget_hook.exe`, and align the example TOML plus README note to the same workspace-root target convention.
- Rationale: In this repository, Cargo outputs shared workspace artifacts under the root `target/` directory. Pointing hooks at a stale per-crate path silently bypasses newer fixes and makes the UI appear inconsistent with the checked-in source.
