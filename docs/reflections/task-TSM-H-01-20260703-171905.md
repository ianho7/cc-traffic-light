- Task: TSM-H-01 把 tray 的 Open Settings 主入口切到 Tauri
- Encountered Problem: 当前 tray 打开链路虽然已经支持 Tauri settings 进程，但仍然依赖环境变量显式开启，默认用户路径还没有真正切换主入口。
- Thought Process: 我没有改动 `handle_tray_action()` 的整体 fallback 结构，只把 `settings_process::tauri_settings_enabled()` 的默认判定翻转为“默认启用 Tauri”，避免在主入口切换时同时重写 tray、Slint 或 Win32 fallback 逻辑。
- Options Considered: 继续保留环境变量灰度；直接删除 Slint 分支；默认走 Tauri，但保留现有失败回退链路。
- Chosen Solution: 更新 `taskbar-widget/src/settings_process.rs`，让未设置 `CC_TRAFFIC_LIGHT_SETTINGS_HOST` 时默认启用 Tauri settings 进程，tray `Open Settings` 因而首先进入 Tauri 路径。
- Rationale: 这样可以最小改动完成主入口切换，同时保留现有 `open_or_focus_tauri_settings() -> Slint -> Win32` 的保守回退链路。
