- Task: TSM-A-05 明确迁移阶段回退策略
- Encountered Problem: 当前默认 settings 已经切到 Slint，若 Tauri 尚未闭环就直接替换入口，风险会落到用户可见主路径上。
- Thought Process: 我把回退门槛写成三项验收条件：真实状态显示、配置写回、即时生效。
- Options Considered: 立刻切 Tauri；长期保留双主入口；先保留 Slint，等 Tauri 达到闭环后再切。
- Chosen Solution: 默认保留 Slint host，直到 Tauri 满足三项门槛。
- Rationale: 这是对当前主链路最保守的替换方式。
