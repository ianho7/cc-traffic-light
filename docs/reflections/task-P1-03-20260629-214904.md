- Task: P1-03 在 `WM_PAINT` 中实现固定背景色与固定文本绘制
- Encountered Problem: `DrawTextW` 在 `windows 0.58` 中接受的是 UTF-16 切片，不是直接的 `PCWSTR` 常量。
- Thought Process: 先接受最朴素的每次绘制时 UTF-16 编码，等任务栏嵌入稳定后再考虑是否值得优化。
- Options Considered: 自己写长期缓存；改用别的绘制 API；在 `paint_window` 中即时编码文本。
- Chosen Solution: 通过 `wide_text()` 把 `TASKBAR WIDGET` 转成 UTF-16 切片，再配合 `FillRect`、`SetBkMode`、`SetTextColor` 和 `DrawTextW` 绘制。
- Rationale: 这满足了 MVP 对“固定文本可见”的要求，同时保持实现最小。
