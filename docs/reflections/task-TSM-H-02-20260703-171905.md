- Task: TSM-H-02 保留 Slint fallback 开关
- Encountered Problem: 默认入口切到 Tauri 后，如果没有一个明确、可控的回退开关，后续稳定性联调会把 fallback 行为和真实故障恢复混在一起。
- Thought Process: 我不引入新的配置文件字段，也不做复杂 runtime 选择器，只复用现有 `CC_TRAFFIC_LIGHT_SETTINGS_HOST` 环境变量，把 `slint` 解释为显式 fallback 模式。
- Options Considered: 不保留显式 fallback；新增单独的 fallback 配置项；沿用现有环境变量并把 `slint` 作为强制回退值。
- Chosen Solution: 在 `settings_process.rs` 中保留 `CC_TRAFFIC_LIGHT_SETTINGS_HOST=slint` 的分流条件，并在架构文档中明确这是 Tauri 稳定性验收前的 fallback 开关。
- Rationale: 这满足了 H-02 对 fallback 开关的要求，也避免把旧 Slint 路径误当成第二默认主入口。
