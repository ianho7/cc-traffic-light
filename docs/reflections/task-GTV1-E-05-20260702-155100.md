- Task: GTV1-E-05 实现后台周期性重试 widget 恢复逻辑
- Encountered Problem: 只有一次性降级还不够；如果 Explorer 刷新、任务栏短暂异常或启动时机不对，软件会永远停在 tray-only，用户必须手动重启才能恢复 widget。
- Thought Process: 重试逻辑应该尽量轻，不引入第二线程或复杂调度，并且继续复用现有主窗口 timer。这样既能和现有轮询 detector 共存，也能把 mount state 暴露给 settings / diagnostics。
- Options Considered: 1. 不做自动恢复，只依赖用户手动重启。2. 新起专门线程负责 attach 重试。3. 在现有主窗口 timer 中按固定间隔做轻量恢复尝试。
- Chosen Solution: 在现有 `WM_TIMER` 路径中增加 `5s` 周期重试：重新 `probe_taskbar`、`attach_to_taskbar`、`position_in_taskbar`，成功后恢复显示并刷新可见性，失败则保持隐藏并标记 `Retrying`。
- Rationale: 复用现有消息循环和 timer 最符合 V1 边界，代码面最小，也不会把恢复机制扩张成新的运行时子系统。
