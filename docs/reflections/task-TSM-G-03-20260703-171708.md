- Task: TSM-G-03 保存成功后立即通知宿主刷新或重载
- Encountered Problem: 前端保存成功后虽然已经会调用 `notify_settings_applied`，但宿主此前只记日志，不会真正触发 detector / snapshot 刷新，因此“即时生效通知路径”还不完整。
- Thought Process: 我沿用现有主窗口 `WM_COMMAND + TRAY_CMD_REFRESH` 路径，不发明新消息类型；只要保存结果里存在实际配置改动，就由宿主在收到 notify 后投递一次 refresh，从而复用现有 `poll_hook_state` 与 settings host 同步逻辑。
- Options Considered: 继续只记录日志；新增一条专门的 Win32 自定义消息；复用当前 refresh 命令路径，在 notify 后立即投递一次刷新。
- Chosen Solution: 在 `taskbar-widget/src/settings_bridge.rs` 中增强 `notify_settings_applied()`，对除 `diagnostics.last_opened_page` 以外的实际配置变更触发一次 `TRAY_CMD_REFRESH` 消息。
- Rationale: 这能最小成本满足“保存成功后立即通知宿主刷新或重载”的要求，也避免引入第二套宿主刷新机制。
