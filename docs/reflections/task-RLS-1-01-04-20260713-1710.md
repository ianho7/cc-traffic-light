# RLS-1-01 至 RLS-1-04：Explorer Recovery 基线、实现与回归

- Encountered Problem: Explorer 重启销毁原 widget HWND 后，host 进程仍存在但不再有可见 widget。
- Thought Process: 先保留旧 release 的 `WM_NCDESTROY` 基线，再使异常销毁进入受控的窗口重建、tray 恢复和 attach retry 路径，并以实际 Explorer 重启验证。
- Chosen Solution: `WM_NCDESTROY` 在非主动退出时重建 HWND、重绑 settings bridge、恢复 timers 和 attach retry；tray state 防止重复注册，并在 Explorer 重启后允许单次 retry。
- Evidence: `%TEMP%\cc-traffic-light-explorer-recovery-20260713-163739.log` 记录 `WM_NCDESTROY → new HWND → tray pending → tray registered → attach`；原 PID 存活，host_count=1。
- Validation: `cargo test -p taskbar-widget --offline`（35 tests）和 `cargo build -p taskbar-widget --release --offline` 通过。
- Remaining Risk: 最终 release bundle 仍需要在 P6 统一归档正常动效截图、manifest 与 gate report；安装/清洁环境验收保持用户接受的 limitation。
