- Task: SSM-C-03 实现统一本地化访问层
- Encountered Problem: settings 和 tray 已经同时需要国际化，如果各自直接读 JSON 或写 match，会很快分叉。
- Thought Process: 把资源读取、locale 推导、字符串 fallback 和 tooltip 拼装全部集中到一个共享模块。
- Options Considered: settings/tray 各自实现文案层；新增共享 `i18n.rs` 统一输出。
- Chosen Solution: 新增共享 i18n 模块，统一提供文本访问、tooltip 组装和 label 映射。
- Rationale: 这保证后续 Settings host、tray、tooltip 和其他用户可见文本都站在同一套资源契约上。
