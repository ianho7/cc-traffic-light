- Task: TSM-I-01 确认 settings_slint.rs 已归档且不再承担默认主入口职责
- Encountered Problem: 先前虽然 tray 默认已切到 Tauri，但宿主仍会初始化 Slint host，并把它挂在 fallback 与事件循环里，无法算真正退场。
- Thought Process: 我从 `main.rs` 入口追到 tray 打开逻辑、`WM_DESTROY` 清理逻辑和宿主事件循环，确认 Slint 仍参与运行时主链路，而不只是历史文件残留。
- Options Considered: 1. 仅保留源码但不再默认调用。2. 保留 Slint 作为显式 fallback。3. 把 Slint 完全移出主链路，仅保留 Win32 fallback。
- Chosen Solution: 删除宿主对 `settings_slint` 的模块引用、初始化、show/update/shutdown 调用，并把默认链路收口为 `Tauri -> Win32 fallback`。
- Rationale: 这样才能明确证明 settings 默认入口已经完全迁到 Tauri，同时不冒险改动 widget、tray 或宿主其他 Win32 主链路。
