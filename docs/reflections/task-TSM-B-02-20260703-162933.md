- Task: TSM-B-02 新建 `crates/shared-core/` 并迁移纯业务类型与配置读写逻辑
- Encountered Problem: 现有 `taskbar-widget` 把配置、状态和 UI/runtime 一起放在单 crate 中，后续 Tauri 和 shared-core 会没有干净落点。
- Thought Process: 我先把最稳定、最纯的三块迁出去：`AppConfig`、`AppStatusSnapshot` 相关类型、`RuntimeContract`，并让宿主 crate 通过旧模块名回导出，避免一次性打碎调用面。
- Options Considered: 直接在 `taskbar-widget` 内新建子模块；一次性迁更多 detector / i18n 代码；只先迁纯业务模型和配置读写逻辑。
- Chosen Solution: 新建 `crates/shared-core/`，迁移纯 Rust 类型与配置文件读写逻辑，并保持 `taskbar-widget` 侧兼容导出。
- Rationale: 这样可以最小扰动地建立共享层，同时严格避免把 Win32 句柄、Slint/Tauri runtime 依赖带进 shared-core。
