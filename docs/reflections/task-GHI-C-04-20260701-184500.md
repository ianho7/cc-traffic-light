- Task: GHI-C-04 implement restore flow
- Encountered Problem: restore 既要支持恢复已有文件，也要支持把“首次安装创建的文件”恢复成不存在状态。
- Thought Process: 单靠备份正文不够，还需要知道 apply 前文件是否存在。
- Options Considered: restore 一律 copy 备份；缺失场景不支持 restore；用 meta 区分 copy-back 和 delete-target。
- Chosen Solution: restore 根据 `originalExisted` 决定恢复文件内容还是删除目标文件。
- Rationale: 这样 restore 才能真实回到 apply 前状态，而不是只回到某个近似状态。
