- Task: GTV1-B-05 Implement the General settings page with autostart and tray-behavior toggles
- Encountered Problem: The settings shell existed, but it was still static. The first interactive page needed to stay pure Win32 and persist values without requiring detector or autostart backend work to be finished first.
- Thought Process: I used the existing AppConfig schema as the source of truth, added page switching to the custom-painted settings window, and implemented click hit-testing only for the General page toggles.
- Options Considered: Add real Win32 checkbox controls; keep the shell static and delay interaction; implement lightweight custom-painted toggle rows backed by AppConfig persistence.
- Chosen Solution: Implement lightweight custom-painted toggle rows backed by AppConfig persistence.
- Rationale: That keeps the pure Win32 shell coherent, proves the config write path, and avoids introducing child-control complexity before the page architecture is settled.
