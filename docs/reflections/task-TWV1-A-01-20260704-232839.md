- Task: TWV1-A-01 Extract background, layout, and state mapping from the old widget paint path
- Encountered Problem: The old widget mixed black background painting, lamp layout, and status mapping directly inside `main.rs`.
- Thought Process: I separated the stable rendering inputs from the Win32 message plumbing first, so transparency changes would not have to mutate detector or tray logic.
- Options Considered: Keep patching the old `WM_PAINT` body; move only the circle draw helper; introduce a dedicated widget render module.
- Chosen Solution: Add a dedicated `widget_render` module and make `main.rs` feed it snapshot plus config.
- Rationale: That was the smallest change that removed black-background coupling while keeping the host architecture intact.
