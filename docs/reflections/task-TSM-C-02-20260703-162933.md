- Task: TSM-C-02 为宿主补独立 settings bridge 模块
- Encountered Problem: `settings_window.rs` 同时承担 fallback UI、配置存储、snapshot 缓存、主窗口 refresh 命令，职责已经不适合继续被新架构复用。
- Thought Process: 我把宿主必须保留的职责单独抽成 `settings_bridge.rs`，只保留配置/状态缓存、写配置、autostart 副作用和 refresh 消息派发。
- Options Considered: 继续把 helper 留在 `settings_window.rs`；直接做 Tauri IPC；先抽一个宿主 bridge，再让现有 Slint/fallback UI 都接它。
- Chosen Solution: 新增 `taskbar-widget/src/settings_bridge.rs`，集中暴露 `current_config`、`current_snapshot`、`update_config`、`toggle_autostart_setting`、`request_manual_refresh_command` 等入口。
- Rationale: 这样 settings backend 首次脱离了具体 UI 文件，后续无论是 Tauri command 还是本地 IPC，都可以直接挂到 bridge 上。
