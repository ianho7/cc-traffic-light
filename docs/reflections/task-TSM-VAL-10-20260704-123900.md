- Task: TSM-VAL-10 连续多次打开/关闭 settings 后，无重复进程、僵尸窗口或状态不同步
- Encountered Problem: 早期因为 loader 错误、窗口标题匹配过窄和宿主强杀收尾，自动验证会在中途误报失败，无法把真正的生命周期错误和脚本误判区分开。
- Thought Process: 先通过分包构建修掉宿主 loader 阻塞，再补足“首次打开只起一个进程”“再次打开复用同一进程”“关闭后退出”“关闭后重新打开生成新进程”这些精确断言。
- Options Considered: 只人工点击；只检查进程数；用脚本对单次打开、复用、关闭后重开三段链路分别断言。
- Chosen Solution: 使用 lifecycle 报告中的 `open_spawned_once`、`reopen_reused_same_process`、`close_exited_process`、`reopen_after_close_spawned` 四个字段作为验收证据。
- Rationale: 这四个字段合起来正对应“没有重复进程、没有僵尸窗口、状态不会在多次打开关闭后漂移”。最新 report 中它们全部为 `true`。
