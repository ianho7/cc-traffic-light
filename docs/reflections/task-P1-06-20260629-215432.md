- Encountered Problem: 纯工具侧虽然已经证明窗口可见且可正常退出，但 checklist 仍要求普通窗口基线在当前机器上被实际确认。
- Thought Process: 先用本地 Win32 API 自动验证窗口两次都能找到、可见、关闭后退出码为 0，再以用户的肉眼确认作为更强证据闭合这个阶段门。
- Options Considered: 只依赖自动化证据；只依赖人工口头反馈；把自动化结果和人工确认合并作为阶段验收。
- Chosen Solution: 结合两轮自动化窗口探测结果与用户“已验证可以打开 TASKBAR WIDGET 窗口”的确认，完成 Phase 1 可见性验收。
- Rationale: 对桌面窗口类任务，自动化和肉眼证据组合比单一证据更可靠。
