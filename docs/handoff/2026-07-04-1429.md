## Session Objective

本轮目标是把 `tauri-settings-migration.md` 里的 Slint 退场阶段真正做完，确认 settings 主链路已经完全迁到 `Tauri`，并补齐对应的构建、生命周期验证和归档文档。

## What Changed

- `taskbar-widget` 宿主不再编译或初始化 `Slint settings`。
- `taskbar-widget/src/main.rs` 已改为纯 Win32 message loop，不再依赖 `slint::run_event_loop_until_quit()`。
- tray `Open Settings` 现在只先尝试 `Tauri`；失败时仅回退到 `settings_window.rs` 的 Win32 极限 fallback。
- `taskbar-widget/Cargo.toml` 已移除 `slint`、`slint-build`、`raw-window-handle` 相关依赖。
- `taskbar-widget/build.rs` 只保留 Windows manifest 嵌入逻辑，不再编译 `ui/settings.slint`。
- 旧 `settings_slint.rs` 与 `ui/settings.slint` 已归档到 `archive/slint-settings/`。
- 根 `package.json` 已补 `build` 脚本，`taskbar-settings-tauri/package.json` 的 `build` 已改为调用 `scripts/build.ps1`，避免 `Volta` wrapper 干扰。
- `pnpm-workspace.yaml` 已把 `allowBuilds.esbuild` 落成 `true`，不再停在占位配置。
- `taskbar-settings-tauri/src/App.tsx` 已补一层前端 locale 字典，页面标题、描述、状态标签、设置项、按钮、保存提示和时间格式都会跟随 `localization.language` 或系统语言切换。
- `diagnostics.last_opened_page` 现在仍会持久化，但不再弹出 `Applied diagnostics.last_opened_page` 这类内部 key 提示。

## Current Architecture

- 默认 settings 主入口：`taskbar-settings-tauri.exe`
- 宿主 fallback：`taskbar-widget/src/settings_window.rs`
- 已归档旧实现：`archive/slint-settings/`
- 宿主事件循环：纯 Win32 message loop
- IPC：named pipe JSON
- 持久化事实源：`%APPDATA%\\CcTrafficLight\\config.json`

## Important Build Notes

- 根级 `pnpm build` 现在可用，但在无 TTY 自动化环境下建议显式带 `CI=true`。
- 本轮为了让 `esbuild` 正常完成 postinstall，执行过一次无沙箱：
  - `pnpm install --config.confirmModulesPurge=false`
- `taskbar-settings-tauri/src-tauri/Cargo.toml` 现在必须保留：
  - `default = ["custom-protocol"]`
  - `custom-protocol = ["tauri/custom-protocol"]`
- 原因：普通 `cargo build -p taskbar-settings-tauri --offline` 如果缺少这个默认 feature，会被 Tauri 当成 `dev` 模式，窗口基地址落到 `devUrl = http://127.0.0.1:1420`，从 `.\target\debug\taskbar-widget.exe` 打开 settings 时就会出现“127.0.0.1 拒绝连接”。
- 本轮已确认最新构建输出不再包含 `cargo:rustc-cfg=dev`，而 `target/debug/build/tauri-*/out/checked_features` 含 `custom-protocol`。
- 宿主验收仍然不要用 `cargo build --workspace --offline`。
- 保持：
  - `cargo build -p taskbar-settings-tauri --offline`
  - `cargo build -p taskbar-widget --offline`
