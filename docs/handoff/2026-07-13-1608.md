## Problems Encountered and Decision Process

### Codex hook `exit code 1`

**Investigation:** 直接回放 hook EXE 可以成功写 state；真实 runner 没有进入 EXE 的诊断。将同事件替换为 `cmd.exe /d /s /c exit 0` 成功，直接 EXE 与 cmd inline 失败，no-space `.cmd` wrapper 成功。

**Solutions considered:**

- 改 state JSON/stdout/Rust 逻辑：排除，直接回放成功。
- 重装/修复 elevated sandbox：`[windows] sandbox = "unelevated"` 是必要运行时绕过，但不解决直接 EXE runner 问题。
- 采用 Windows `.cmd` wrapper：已选择并验证。

**Implication:** `commandWindows` 必须继续调用 `cmd.exe ... call wrapper.cmd`；不要改回直接 EXE。

### Explorer recovery

**Investigation:** 正常 host 带 `TASKBAR_MVP_RUNTIME_LOG_FILE` 运行后重启 Explorer。host PID 存活，用户看见 widget 消失；日志终止于 widget `WM_NCDESTROY`。

**Final decision:** 记录为产品缺陷，不将“手工重启 host 后恢复”误当成自动恢复。后续需设计窗口重建/重新挂载生命周期。
