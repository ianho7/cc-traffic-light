# 06. 验证与调试

## 1. 验证目标

你不是在验证“界面好不好看”。

你在验证：

- 宿主句柄能不能找到
- 子窗口能不能挂进去
- 挂进去后能不能定位出来
- 这个窗口能不能自己画内容

## 2. 最小验证清单

### 检查 1：普通窗口可见

预期：

- 启动程序后能看到一个普通窗口
- 窗口里有固定文本

如果失败，先不要碰任务栏逻辑。

### 检查 2：任务栏句柄可获取

预期：

- 日志打印出非空 `Shell_TrayWnd`

如果失败：

- 先检查 API 调用和字符串
- 再检查权限和运行环境

### 检查 3：`SetParent` 成功

预期：

- 调用返回成功
- 没有明显错误码

如果失败：

- 打印 `GetLastError`
- 检查传入父句柄是否有效

### 检查 4：窗口不再是普通浮窗

预期：

- 窗口行为明显依附任务栏
- 至少不是单独悬浮在桌面中间

### 检查 5：任务栏中出现固定文本模块

预期：

- 肉眼可见一个小矩形区域
- 区域中有固定文本

## 3. 推荐的调试输出

每次运行至少打印：

- 当前目标系统信息
- 任务栏句柄
- 父窗口句柄
- `SetParent` 结果
- `GetWindowRect` 结果
- 最终 `MoveWindow` 的坐标

## 4. 最常见的失败情况

### 情况 1：程序运行但什么都看不到

可能原因：

- `MoveWindow` 坐标错误
- 窗口已经挂进去，但跑到不可见区域
- 宽高为 0

处理方法：

- 给窗口加醒目的背景色
- 先放大尺寸
- 先用保守固定坐标

### 情况 2：还是普通浮窗

可能原因：

- `SetParent` 失败
- 传错父窗口
- 调用时机不对

处理方法：

- 打印 `SetParent` 返回值
- 记录错误码

### 情况 3：任务栏里有空位，但没有文字

可能原因：

- `WM_PAINT` 没触发
- `BeginPaint/EndPaint` 使用错误
- 文本颜色和背景色一样

处理方法：

- 先画纯色背景
- 再画文本

### 情况 4：模块闪一下就消失

可能原因：

- 消息循环提前退出
- 窗口被系统重绘覆盖
- 位置计算逻辑在别处又把它移走

处理方法：

- 保持逻辑简单
- 不要过早加入定时器

## 5. 推荐的最小手动测试

每次改完执行：

1. `cargo run`
2. 观察是否出现模块
3. 关闭程序
4. 再次运行
5. 观察结果是否一致

如果两次结果不一致，先修稳定性，不要加功能。

## 6. 哪些问题现在可以接受

MVP 阶段可以接受：

- 模块位置不够优雅
- 模块背景不是透明
- 只支持一种任务栏样式
- 关闭后没有恢复所有系统细节

只要核心目标成立，就不要因为这些问题阻塞进度。
